Translate Python code into optimized Cython (.pyx) that compiles and runs faster.
Use the evaluate_cython tool iteratively:
1. Write initial Cython code and evaluate it.
2. Read the compilation errors, annotation hints, test failures, and speedup.
3. Fix issues and evaluate again.
4. Repeat until: all tests pass, annotation score > 0.9, and speedup is maximized.

## TOOL USAGE

The evaluate_cython tool accepts ONE argument: `code` (the complete .pyx source).
Do NOT pass `func_name`, `python_code`, `cython_code`, or any other arguments.

## FUNCTION SIGNATURE (CRITICAL)

The benchmark harness calls your function with NO arguments for warmup, then with arguments for testing.
- CORRECT: `def func(n=1000):` — untyped default, works for both calls
- CORRECT: `def func(n=None):` with `if n is None: n = 1000` inside
- WRONG: `def func(int n=1000):` — typed default fails as positional arg
- WRONG: `cpdef func(int n=1000):` — cpdef typed defaults fail warmup call

## COMPILER DIRECTIVES

Always include at top of file:
```
# cython: language_level=3, boundscheck=False, wraparound=False, cdivision=True
```

## CYTHON SYNTAX RULES

1. ALL `cdef` declarations at TOP of function, before any executable code:
   ```cython
   cdef int i, j, k
   cdef double total = 0.0
   ```
   NEVER inside loops or after statements — causes "cdef statement not allowed here".

2. C array init uses braces `{}` not brackets `[]` inside functions:
   - WRONG: `cdef int arr[4] = [1, 2, 3, 4]`
   - RIGHT: `cdef int arr[4] = {1, 2, 3, 4}`

3. Use C standard library for performance:
   - `from libc.math cimport cos, sin, sqrt, fabs, M_PI`
   - `from libc.stdlib cimport malloc, free, qsort`
   - `from libc.string cimport memset, memcpy, memcmp`
   - `from libc.stdint cimport uint32_t, uint64_t`

4. Memory allocation — always check NULL and free:
   ```cython
   cdef int *buf = <int *>malloc(n * sizeof(int))
   if buf == NULL:
       raise MemoryError()
   # ... use buf ...
   free(buf)
   ```

5. `nogil` blocks require pure C — no Python objects:
   - Declare helpers as `cdef inline int func(int x) nogil:`
   - Convert Python args to C types BEFORE the nogil block
   - Use `noexcept` on nogil functions that don't raise

6. No stray `}` after function definitions.

7. No unused imports (`import cython` when not needed).

## COMMON ERRORS AND FIXES

- "cdef statement not allowed here" → move declarations to top of function
- "Calling gil-requiring function not allowed without gil" → add `nogil` to helper signature
- "Coercion from Python not allowed without the GIL" → cast Python arg to C type before nogil block
- "missing 1 required positional argument" → use `def func(n=1000):` not typed default
- "Expected an identifier or literal" → check for stray `}` after functions
- "Syntax error in C variable declaration" → move array init inside function

## OPTIMIZATION PATTERNS

### Replace Python lists with C arrays
```cython
cdef int *dp = <int *>malloc((n + 1) * sizeof(int))
memset(dp, 0, (n + 1) * sizeof(int))
# ... use dp[i] instead of list[i] ...
free(dp)
```

### Sorting with qsort
```cython
cdef int cmp_func(const void *a, const void *b) noexcept nogil:
    return (<int *>a)[0] - (<int *>b)[0]
qsort(arr, n, sizeof(int), cmp_func)
```

### C math instead of Python math
```cython
from libc.math cimport sin, cos, sqrt, M_PI
# Direct C calls, no Python overhead
```

### Precompute outside loops
```cython
cdef double inv_n = 1.0 / n
for i in range(n):
    result += values[i] * inv_n  # multiply instead of divide
```

## ITERATIVE PROCESS

1. **Write**: Translate Python to Cython with basic `cdef` types and correct algorithm
2. **Evaluate**: Call evaluate_cython — read ALL feedback sections
3. **Fix compilation errors** first (nothing else matters until it compiles)
4. **Fix test failures** next (correctness before performance)
5. **Improve annotation score** — add more `cdef`, replace Python constructs with C
6. **Maximize speedup** — use malloc arrays, nogil, precompute constants
7. **Evaluate again** until all criteria are met
